# Contributing to Vantigo
Thank you for considering contributing to Vantigo! Your input is invaluable in making this project better for everyone. Whether you are a seasoned developer or a beginner, we appreciate your help.

## How to Contribute

### 1. Fork the Repository
Start by forking the [Vantigo repository](https://github.com/kraft-tech/vantigo) to your GitHub account. This allows you to freely make changes without affecting the original codebase.

### 2. Create a New Branch
Before making any changes, create a new branch for your feature or bug fix:

```bash
git checkout -b feature/my-new-feature
```
or for a bug fix:
```bash
git checkout -b bugfix/my-bug-fix
```

### 3. Make Your Changes
Make the necessary changes in your branch. Be sure to write clear, concise commit messages that explain your modifications.

### 4. Test Your Changes
If you’ve added new features or made bug fixes, please ensure that you thoroughly test your changes. This helps maintain the quality and integrity of the project.

### 5. Submit a Pull Request
Once you're satisfied with your changes, push your branch to your GitHub fork:
```bash
git push origin feature/my-new-feature
```

Then, go to the original Vantigo repository and create a new pull request. Provide a clear description of your changes and any relevant context to help us review your contribution.

## Reporting Issues
If you encounter bugs or have suggestions for new features, please report them in the Issues  section of the repository.

## Questions?
If you have any questions or need further clarification, feel free to reach out by opening an issue or contacting a maintainer directly.

Thank you for contributing to Vantigo! Together, we can build a powerful business management system that supports users around the globe.
